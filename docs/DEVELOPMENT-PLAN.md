# 安牛项目开发计划

**Summary**: 详细的开发步骤规划，每一步都有明确的规则、验收标准和提交要求。

**Last updated**: 2026-05-09

---

## 开发原则

1. **先规则后执行**：每一步开发前先明确技术规范和实现方案
2. **严格按PRD**：所有功能以product-requirements.md为准
3. **小步快跑**：每步都是可运行、可测试的完整功能
4. **测试验证**：每步完成后进行测试，确保符合预期
5. **及时提交**：每步完成后写清晰的commit message并提交

## 提交规范

### Commit Message格式

```
<type>: <subject>

<body>

<footer>
```

**type类型**:
- `feat`: 新功能
- `fix`: 修复bug
- `docs`: 文档更新
- `style`: 代码格式
- `refactor`: 重构
- `test`: 测试相关
- `chore`: 构建/工具链

**示例**:
```
feat: 实现侧边栏导航框架

- 创建Sidebar组件
- 实现5个导航标签
- 配置路由切换
- 添加底部状态栏

Closes #1
```

## 风险控制

### 技术风险
- Tauri 2.0文档不完善 → 参考官方示例和GitHub issues
- Tantivy中文分词 → 使用jieba-rs
- python-pptx复杂布局 → 提供多种模板

### 时间风险
- 每步预留20%缓冲时间
- 复杂功能拆分为更小的步骤
- 优先实现核心功能
