## 一、产品概述

### 1.1 产品名称
**安牛** - 企业安全知识库助手

### 1.2 产品定位
面向企业安全管理人员的个人知识库桌面客户端，实现文档智能化沉淀、结构化知识管理、智能问答与检索、培训材料自动生成。

### 1.3 核心价值
- **自动化知识沉淀**：上传文档 → AI自动解析生成结构化Wiki
- **双模式查询**：知识问答（AI理解）+ 原文检索（精确匹配）
- **多知识库管理**：支持创建多个独立知识库，按主题分类
- **多模型配置**：支持DeepSeek/OpenAI/通义千问等多种模型
- **知识可追溯**：每个知识页面都有明确来源，删除文档联动清理
- **培训材料生成**：从知识库/文档自动生成培训PPT，支持多来源组合

### 1.4 目标用户
- 企业安全管理人员
- 法务合规人员
- 技术文档管理者
- 需要管理专业知识文档的个人用户

## 二、整体交互设计

### 2.1 侧边栏导航

```
┌──────────────┐
│  🤖 对话      │ ← 智能问答，可引用多个知识库
│  🔍 检索      │ ← 原文模糊搜索，可切换知识库
│  📚 知识库    │ ← 知识库管理（增删知识库/文档）
│  📊 培训      │ ← 培训材料生成（PPT/文档）
│  ⚙️ 设置      │ ← 模型配置、系统设置
└──────────────┘
```

### 2.2 底部状态栏

```
┌─────────────────────────────────────────────────────────┐
│  当前知识库: 港口安全知识库 [切换▼]  |  模型: V3 Flash [▼] │
└─────────────────────────────────────────────────────────┘
```

### 2.3 全局PPT生成入口

PPT生成作为核心功能，提供**3个入口**：

| 入口位置 | 触发场景 | 预选内容 |
|---------|---------|---------|
| **侧边栏「培训」标签** | 独立使用培训生成功能 | 用户手动选择 |
| **知识库页面** | 管理某个知识库时快速生成 | 当前知识库的所有文档 |
| **对话页面** | 问答过程中想生成培训材料 | 当前对话引用的知识库 |

#### 核心交互

1. **创建知识库**
   - 点击"+ 创建知识库"
   - 输入知识库名称
   - 自动生成ID和目录结构
   - 初始化空wiki/index.md

2. **删除知识库**
   - 点击"删除知识库"
   - 二次确认："此操作将删除整个知识库，包括所有文档和Wiki页面"
   - 确认后删除整个目录

3. **添加文档**
   - 点击"+ 添加文档"
   - 选择文件（PDF/Word/Markdown）
   - 自动解析生成Wiki页面
   - 显示解析进度

4. **删除文档**
   - 点击文档的"删除"按钮
   - 系统扫描相关Wiki页面
   - 提示："删除此文档将同时删除12个Wiki页面，其中3个被其他文档引用"
   - 用户选择：
     - 仅删除文档，保留Wiki页面
     - 删除文档和所有相关Wiki页面
     - 取消删除

5. **快速生成PPT**
   - 点击知识库卡片内的"生成PPT"
   - 自动跳转到「培训」页面
   - 预选当前知识库的所有文档
   - 用户可直接配置生成参数或调整选择

#### 核心交互

1. **三种数据来源**
   - **从知识库生成**：选择一个或多个知识库，提取所有Wiki页面内容

   - 支持同时选择多个知识库
   - 支持跨知识库选择文档

   - 幻灯片数量：控制PPT页数
   - 内容侧重点：选择需要重点展示的内容类型
   - PPT模板：提供多种预设模板

   - 确认后才生成PPTX文件

5. **生成进度展示**
   - 分步骤显示进度（提取知识点 → 生成内容 → 生成PPTX）
   - 实时进度条
   - 预计剩余时间

6. **生成后操作**
   - 下载PPTX文件
   - 打开所在文件夹
   - 预览PPT（内置阅读器）
   - 重新生成（调整参数后）

## 五、核心数据流

### 5.1 文档上传与解析

```
用户选择知识库 → 添加文档 → 保存到raw/ → 
调用LLM解析 → 生成Wiki页面 → 更新index.md → 
更新文档追踪.json → 更新search_index → 完成
```

### 5.2 知识问答

```
用户选择知识库（可多个）→ 输入问题 → 
检索选中知识库的index.md → 读取相关Wiki页面 → 
调用选定模型生成答案 → 显示答案 + 引用来源
```

### 5.3 原文检索

```
用户选择搜索范围 → 输入关键词 → 
查询search_index → 返回匹配结果（含高亮）→ 
用户点击结果 → 读取原文对应页 → 展示原文 + 高亮
```

### 5.4 文档删除

```
用户选择删除文档 → 读取文档追踪.json → 
识别相关Wiki页面 → 检查交叉引用 → 
提示用户风险 → 用户确认 → 
删除Wiki页面 → 更新index.md → 
更新search_index → 删除raw/文档 → 完成
```

### 5.5 PPT生成（多来源支持）

```
用户选择数据来源：
  选项A：选择一个或多个知识库 → 读取Wiki页面
  选项B：选择一个或多个已上传文档 → 解析文档
  选项C：上传新文档 → 解析文档
  ↓
提取核心知识点
  ↓
调用LLM生成PPT大纲
  ↓
用户确认/编辑大纲
  ↓
使用python-pptx生成PPTX
  ↓
保存到output/ → 提供下载
```

## 八、关键设计决策

### 8.1 为什么采用侧边栏4标签设计？

**优势**：
- 功能分区清晰：对话、检索、管理、设置各司其职
- 扩展性强：后续可增加"绘图"、"数据分析"等标签
- 符合桌面应用习惯：类似Cherry Studio、Notion等
- 用户认知成本低：图标 + 文字，一目了然

### 8.2 为什么支持多知识库引用？

**场景**：
- 用户可能同时需要查询"港口安全知识"和"安全生产制度"
- 不同知识库可能包含互补信息
- 避免频繁切换知识库的繁琐操作

**实现**：
- 对话页面可同时勾选多个知识库
- 检索时合并多个知识库的结果
- 引用来源标注所属知识库

### 8.3 为什么文档删除需要联动清理？

**原因**：
- 一个文档会生成10-15个Wiki页面
- 如果不删除，会产生"孤儿页面"
- 长期积累会导致知识库臃肿
- 保持知识库的整洁和可维护性

**风险控制**：
- 二次确认机制
- 显示影响范围（多少页面会被删除）
- 提示交叉引用风险
- 提供"仅删文档保留页面"选项

### 8.4 为什么PPT生成采用Skill系统？

**优势**：
- 可扩展：后续可增加“生成Word报告”、“生成思维导图”等Skill
- 解耦：PPT生成逻辑独立于核心功能
- 可配置：用户可自定义Prompt、模板
- 可替换：可升级PPT生成引擎而不影响其他功能

### 8.5 为什么培训页面作为独立侧边栏标签？

**原因**：
1. **功能独立性强**：培训材料生成与对话/检索/知识库管理都不同
2. **使用频率高**：企业用户经常需要生成培训材料
3. **操作流程长**：3步骤向导（选择来源 → 配置参数 → 预览生成）
4. **入口多样化**：虽然有3个入口，但主入口应该明确

**3个入口设计**：
- **主入口**：侧边栏「培训」标签 → 完整功能，自由选择来源
- **快捷入口1**：知识库页面「生成PPT」按钮 → 预选当前知识库
- **快捷入口2**：对话页面快捷操作 → 基于当前对话上下文

**用户场景**：
```
场景1：从知识库生成
  用户在知识库页面 → 点击“生成PPT” → 
  跳转到培训页面 → 已预选当前知识库 → 
  配置参数 → 生成

场景2：从多个知识库组合生成
  用户点击侧边栏“培训” → 选择“从知识库生成” → 
  勾选多个知识库 → 配置参数 → 生成

场景3：从已上传文档生成
  用户点击侧边栏“培训” → 选择“从已上传文档生成” → 
  搜索并选择文档 → 配置参数 → 生成

场景4：从新上传文档生成
  用户点击侧边栏“培训” → 选择“从新上传文档生成” → 
  上传文件 → 配置参数 → 生成
```

### 8.6 为什么PPT生成支持多来源？

**场景需求**：
1. **跨知识库培训**：需要结合“港口安全知识”和“安全生产制度”生成综合培训
2. **特定文档培训**：只想基于某几个重要文档生成培训，不需要整个知识库
3. **临时培训**：手头有新文档，想立即生成培训材料，不需要先入库

**技术实现**：
- 从知识库生成：直接读取Wiki页面（快速，已结构化）
- 从已上传文档生成：现场解析文档（较慢，但灵活）
- 从新上传文档生成：上传 + 解析 + 生成（最慢，但最灵活）

**优势**：
- 满足不同场景需求
- 提高使用灵活性
- 降低使用门槛（不需要先创建知识库也能生成PPT）

## 九、风险评估

| 风险 | 影响 | 概率 | 缓解措施 |
|------|------|------|----------|
| 多知识库切换性能问题 | 中 | 低 | 懒加载、缓存机制 |
| 文档删除误操作 | 高 | 中 | 二次确认、回收站机制 |
| PPT生成质量不稳定 | 中 | 中 | 提供大纲编辑、多模板 |
| 交叉引用处理复杂 | 高 | 中 | 来源追踪系统、风险提示 |
| 大知识库加载慢 | 中 | 低 | 分页、懒加载、索引优化 |
