## 三、功能路径正确性评估

### ✅ 正确的地方

1. **API契约设计完善**
   - 前后端交互协议明确 (ApiResponse统一、字段映射完整)
   - 参数结构合理 (ChatRequest/SearchRequest/TrainingConfig完整)
   
2. **前端框架预留到位**
   - 5 个页面框架 (Chat/Search/KnowledgeBase/Training/Settings)
   - 状态管理框架 (useApp Context)
   - 组件库集成完整 (shadcn/ui)

3. **后端接口路由完整**
   - 所有业务端点已定义 (知识库/文档/Wiki/对话/检索/培训)
   - 错误处理框框架有 (HTTPException)

### ⚠️ 有问题的地方

1. **核心业务逻辑缺失** (最严重)
   - `wiki_service.parse_document()` 只有路径框架，无解析实现
   - `chat_service._find_related_pages()` 缺失关键词匹配逻辑
   - `training_service.generate_outline()` 缺失 LLM 调用集成
   - `wiki_service.lint_wiki()` 缺失检查规则实现

2. **LLM 集成不完整**
   - `llm_service.py` 有框架但缺具体实现
   - `AGENTS.md` 定义了但仅在 wiki_service 使用，应全局应用

3. **数据持久化逻辑缺失**
   - 文档如何存储/加载
   - Wiki 如何序列化/反序列化
   - 文档追踪如何更新

4. **前端状态同步不完整**
   - 知识库切换未触发全局状态更新
   - 模型切换状态不持久化
   - ChatPage 的异步流程管理有缺陷

### 🔴 严重偏离需求的地方

1. **文档解析管道未实现**
   - PRD 要求 PDF/Word/Markdown 自动解析生成 Wiki
   - 当前仅有上传接口，无解析逻辑

2. **Wiki Lint 功能缺失**
   - PRD 明确要求检测：格式/断链/孤儿页/缺失来源
   - 当前完全未实现

3. **PPT 生成功能缺失**
   - PRD 作为核心功能之一
   - 目前仅有 UI 框架，无逻辑实现

4. **原文阅读器缺失**
   - PRD 明确要求翻页+高亮+导出
   - 当前完全未实现

## 四、后续开发建议

### 优先级分层

#### 🔴 P0 - 立即做（阻塞功能）

1. **实现文档解析管道** (3-5天)
   ```python
   # backend/services/wiki.py
   async def parse_document(kb_id: str, doc_id: str):
       1. 读取 raw/ 中的文件
       2. 使用 PyPDF/python-docx/markdown 提取文本
       3. 调用 LLM (注入 AGENTS.md) 生成 Wiki 结构
       4. 保存到 wiki/ 目录
       5. 更新文档追踪.json
       6. 更新 KB meta.json 统计
   ```

2. **修复异步流程管理** (1-2天)
   - ChatPage: 改为 async/await，使用 AbortController
   - 添加错误边界与重试机制
   - 流式中断恢复

3. **实现关键词检索** (2-3天)
   ```python
   # backend/services/chat.py
   _find_related_pages():
       - 对 kb 的 wiki/index.md 分词
       - 计算问题与文档相似度
       - 返回 top-K 相关页面
   ```

#### 🟡 P1 - 1周内做（核心功能）

4. **实现 Wiki Lint 检查** (3-4天)
   ```python
   # backend/services/wiki.py
   lint_wiki():
       - 扫描所有 wiki/*.md 文件
       - 检查格式 (frontmatter/标题结构)
       - 检查 [[链接]] 是否有效 (断链检测)
       - 识别未被文档引用的页面 (孤儿页)
       - 验证 sources 字段是否完整 (缺失来源)
   ```

5. **实现搜索高亮+分组** (2-3天)
   - SearchPage 展示结果分组 (按知识库/文档)
   - 高亮关键词位置
   - 分页加载

6. **修复配置同步机制** (1-2天)
   - 知识库切换时更新全局状态
   - 模型切换时保存到 config.json
   - 状态变化时通知所有消费端

#### 🟢 P2 - 2周内做（增强功能）

7. **实现 PPT 生成** (4-5天)
   - 参数化生成 PPT 大纲 (LLM)
   - 使用 python-pptx 生成 PPTX 文件
   - 支持模板选择
   - 进度条展示

8. **实现原文阅读器** (3-4天)
   - PDF 翻页查看 (pdfjs 或 react-pdf)
   - 关键词高亮
   - 导出功能

#### 🔵 P3 - 1月内做（优化项）

9. **搜索性能优化**
   - 集成 Whoosh 或 Elasticsearch
   - 构建倒排索引
   - 支持模糊匹配

10. **并发保护**
    - config.json 写入加锁
    - 文档解析队列化 (celery/rq)

11. **安全加固**
    - API Key 加密存储
    - 用户认证 (可选)
